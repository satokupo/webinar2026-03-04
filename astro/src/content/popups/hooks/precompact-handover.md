# precompact-handover（自動ハンドオーバー生成）

## 何のためのフックか

コンテキストの自動圧縮（auto compact）が発動する直前に、引き継ぎファイルを自動生成する PreCompact フック。
ユーザーが明示的に圧縮を指示しなくても（自動圧縮でも）、文脈が失われないようにするセーフティネット。

## やっていること

1. transcript（JSONL）をパースして、初期タスク・直近の会話・変更ファイル・プランファイルを抽出
2. Sonnet を `claude --print` で呼び出し、抽出データからニュアンスを含む高品質な引き継ぎ文を生成
3. Sonnet 呼び出しが失敗した場合は、機械パースの結果をそのままフォールバックとして保存
4. `.claude/temp/{session_id}_autocompact.md` として保存

## 工夫したこと

- **Sonnet 呼び出し + フォールバック**: まず Sonnet で高品質な引き継ぎ文を生成し、失敗時は機械パース結果にフォールバック。タイムアウト・出力空・キーセクション欠落を検知して切り替える
- **スキル生成との競合回避**: context-handover スキルが生成した `_handover.md` が既に存在する場合、autocompact の生成をスキップ。二重生成を防ぐ
- **タイムアウトガード**: JSONL パースに15秒、Sonnet 呼び出しに45秒のタイムアウトを設定。巨大ログでもフックがハングしない
- **リングバッファ**: 直近10ターン分だけを保持し、長大な会話でもメモリを圧迫しない
