# send-continue（自動続行）

## 何のためのフックか

コンパクション完了後、AIに「続行してください」というメッセージを自動送信する補助スクリプト。
sessionstart-load-handover からバックグラウンドプロセスとして起動される。

ユーザーが圧縮後にわざわざ「続きをやって」と打つ手間をなくす。

## やっていること

1. Phase 1: tmux の capture-pane で画面を監視し、"Compacted" 文字列でコンパクション完了を検知
2. Phase 2: "❯" マーカーで入力受付可能を検知
3. 検知後、tmux send-keys で「引き継ぎファイルの内容に基づいて、コンパクション前の作業をそのまま続行してください。」を送信

## 工夫したこと

- **二段階検知**: 「コンパクション完了」と「入力受付可能」を別々に検知。完了直後はまだ入力を受け付けていないことがあるため、両方の条件が揃ってから送信する
- **タイムアウト設計**: Phase 1 に30秒、Phase 2 に15秒のタイムアウト。検知できなければ静かに終了し、ユーザーの手動入力に任せる
- **tmux 専用**: tmux の send-keys を使うため tmux 環境限定。非 tmux ではそもそも起動されない（sessionstart-load-handover 側で判定）
