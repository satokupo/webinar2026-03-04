# impl-checker（実装完了確認エージェント）

## 何のためのエージェントか

プラン通りに実装が完了したかをチェックするエージェント。
planmode-rules の実装完了後に呼ばれ、プランファイルの内容と実際の git diff/status を照合して、漏れや計画外の変更を検出する。

Agent ツールの `subagent_type: "impl-checker"` として定義されている。

## やっていること

1. プランファイルからファイル操作セクション（どのファイルに何をする予定か）を抽出
2. git diff/status で実際の変更内容を取得
3. プランと実際の差分を意味的に照合
4. 最終レポート出力: ✅実装済み / ❌未実装 / ⚠️計画外変更

## 工夫したこと

- **Sonnet サブエージェント委任**: 変更ファイル数が6以上の大規模な変更では、Sonnet サブエージェントに照合作業を委任して処理を効率化
- **意味的判定**: 単純なファイル名マッチではなく、「プランの意図に対して実際の変更が合致しているか」を意味レベルで判定する
- **計画外変更の検出**: 実装済み/未実装だけでなく、プランにない変更（⚠️）も検出することで、意図しない副作用を発見できる
