# git-commit-rules（コミットルール）

## 何のためのスキルか

git コミット時のメッセージ形式、確認フロー、push ルールを定めた Single Source of Truth。
planmode-rules（実装完了後のコミット）と save-chatlog（ログ保存後のコミット）から呼ばれる。

## やっていること

1. コミットメッセージの形式を規定（日本語、プレフィックス付き、リスト形式の変更説明）
2. Phase 1〜4 のコミットフローを定義（対象の特定 → ステージング → メッセージ作成 → 実行）
3. push 時のルール管理（リモート有無の判定、settings.json の ask ルールとの連携）

## 工夫したこと

- **文脈に応じたフロー分岐**: コミット対象が明確な場合（プラン実行後等）と不明確な場合（`/commit` 単独呼び出し）で Phase 1 の手順を分岐させている
