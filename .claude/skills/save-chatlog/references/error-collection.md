# エラー/課題ログ生成手順

save コマンドの実行後、stderr にエラー収集結果が出力される（検出ゼロ件でも出力される）。
以下の手順に従ってエラー/課題を収集し、ログを生成する。

## 処理フロー

### 1. 機械的収集結果の確認

stderr に出力されたパターンマッチ結果を確認する。各エラーには `type`（traceback, exit_code, permission_denied, error）と内容が記載されている。

stderr の `Chatlog path:` からチャットログパスを取得し、エラーログに含める。

### 2. エラーログ生成

機械的収集結果に基づき、エラーログを生成する。

**エラーログのフォーマット**:

```
# エラー/課題ログ — YYYY-MM-DD

session_id: <セッションID>
chatlog_path: <チャットログのファイルパス（stderr の Chatlog path: から取得）>

## エラー/課題 #1

- **目的**: 何をしようとしていたか
- **期待**: 何が起きるはずだったか
- **実際**: 何が起きたか（エラーメッセージ含む）
- **コンテキスト**: どのスキル/ツール/ファイルが関係していたか
- **対応候補**: スキル修正 / CLAUDE.md追記 / コード修正 / etc.

---
## 対応完了時

すべての問題が解決したら、以下を実行する:
1. このファイルをアーカイブに移動する
2. ルールに従ってコミットコメントを作成する
3. 作成したコミットコメントをユーザーに提示し
   「この内容でコミットしていいですか？」と確認する
4. 承認後、コミットを実行する
```

検出結果がゼロ件の場合は、エラーログを生成せずスキップする。

### 3. ファイルの書き出し

Python が stderr に出力したパスをそのまま使用する:

```bash
python3 .claude/skills/save-chatlog/chatlog_utils.py write \
  "<Python が出力したパス>" "$(cat <<'ERRLOG'
<生成したマークダウン内容>
ERRLOG
)"
```

### 4. 通知

エラーがある場合のみ、以下の形式で通知する:

```
⚠️ エラーログを保存しました: .claude/context/issues/YYYYMMDD-HHMMSS_error-log.md（N件の課題）
```

## 注意事項

- **タイムスタンプは Python が生成したものを使用する**。LLM がタイムスタンプを独自に生成してはいけない
- エラーログのファイルパスも Python の stderr 出力をそのまま使う
- 対象は **セッション中に発生した他のスキル・フロー・ツールのエラー** であり、save-chatlog 自体のエラーではない
- 対応完了後はファイルをアーカイブに移動し、コミットフローへ進む（削除しない）
- **error-collection.md 内から他の references ファイルへの読み込み指示を書かないこと**（ファイル読み込み制御の一元化ルール）
