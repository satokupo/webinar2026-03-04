# リポジトリ外書き込み

## 発動条件

以下のいずれかに該当する場合に起動する:
- プランのファイル操作・検証手順・スクリプト実行で、現在のリポジトリ外かつ dotfiles（`~/dotfiles/`, `~/.claude/`）外のパス（`/tmp/`, `/var/` 等）へのファイル作成・変更・削除が含まれている

判断に迷う場合は起動する側に倒す。

## 検証内容

プランに以下の問題がないかを検証する:

1. **リポジトリ外書き込みの特定**: プラン内のファイルパス（ファイル操作セクション、検証方法セクション、コード例内のパス等）を走査し、現在のリポジトリ外かつ dotfiles 外への書き込み操作が含まれていないか
2. **ユーザー許可の確認**: リポジトリ外書き込みが検出された場合、会話ログ（conversation.md）にてユーザーが AskUserQuestion を通じて明示的に許可しているかを確認する
3. **未許可の指摘**: 許可が確認できない場合は指摘する

## 検証プロンプト

以下を検証エージェントのプロンプトに含める:

「プラン内のファイルパス（ファイル操作セクション、検証方法セクション、コード例内のパス等）を走査し、現在のリポジトリ外かつ dotfiles 外への書き込み操作がないか確認せよ。検出した場合、conversation.md 内でユーザーが AskUserQuestion を通じて明示的に許可しているか確認せよ。許可が見当たらなければ『リポジトリ外書き込み（対象パス: ...）にユーザー許可が確認できない』と指摘せよ」
