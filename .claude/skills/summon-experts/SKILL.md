# 専門家集団召喚スキル

ユーザーの課題について、複数の専門家ロールによるパネルディスカッションを行い、多面的な検討を経て提案を導く。

## 重要な設計原則

- **チャット上に直接出力する**: サブエージェントやファイルには出力しない
- **小出し出力**: 議題ごとに議論結果を出力する
- **段階的な承認**: Phase 1 → 2 → 3 → 4 と進み、各フェーズでユーザーが修正・承認できる

## フロー

スキル発動時、`references/discussion-flow.md` を読み込み、以下のフローに従う:

1. **Phase 1**: 課題分析（YAML整理）+ 専門家ロール提案（3〜5人）→ ユーザー承認
2. **Phase 2**: フルキャラクター設定の展開 + タブリネーム → ユーザー承認
3. **Phase 3**: パネルディスカッション（全議題を1ターンで自動進行）
4. **Phase 4**: 最終提案のまとめ → 次アクション提案（`planmode-rules` へ接続）

## 全体を通しての注意事項

- **議論の出力にサブエージェントは使わない**: 議論テキストはメインコンテキストが直接出力する
- **事実調査にはサブエージェント（Task ツール）を使用できる**: ドキュメント精読が必要な調査はサブエージェントに委任し、要点のみ返してもらう（コンテキスト保護のため）
- **ファイル出力しない**: 議論はすべてチャット上にテキスト出力する
- Phase間の遷移は、ユーザーの承認を待ってから行う
- **矛盾・課題の先送り禁止**: 議論中に発見された矛盾は必ず解消するか、ユーザー承認のもとログを残す（詳細は discussion-flow.md 参照）
