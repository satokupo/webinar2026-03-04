# 自動チェック（一次情報源）

skill-design-guide は起動時に以下の API で自動的にナレッジの変更をチェックする。
実行フローの詳細は `auto-update.md` および `SKILL.md` の「起動時ナレッジ更新チェック」を参照。

- **GitHub Releases API**: `gh api repos/anthropics/claude-code/releases`
  - "skill" キーワードでフィルタし、スキル関連リリースのみ検知
  - 前回チェック日（update-log.yaml の last_checked）以降のリリースが対象
- **補助情報源**: `gh api repos/anthropics/skills/commits`
  - 公式スキルサンプルリポジトリの変更確認（必要に応じて）

---

# 手動チェック（ユーザー指示時のみ）

以下は**手動チェック**の情報源。自動チェック（上記 GitHub Releases API）とは別に、
ユーザーが「最新情報もチェックして」「ドキュメントも確認して」と明示的に指示した場合のみ参照する。

---

## 参照先 URL 一覧

### Anthropic 公式ドキュメント

- **Claude Code スキルドキュメント**: https://docs.anthropic.com/en/docs/claude-code/skills
  - スキルの公式仕様、フロントマターの定義、ベストプラクティス
  - 確認ポイント: フロントマターのフィールド定義に変更がないか、新しいフィールドが追加されていないか

- **Claude Code ドキュメント全体**: https://docs.anthropic.com/en/docs/claude-code/overview
  - Claude Code の全体像、ツールの使い方、設定方法
  - 確認ポイント: スキルに影響する新機能やツールの変更がないか

### GitHub リポジトリ

- **anthropics/skills**: https://github.com/anthropics/skills
  - Anthropic 公式のスキルサンプル集
  - 確認ポイント: 新しいスキルパターンやベストプラクティスの更新がないか

---

## 参照のトリガー条件

以下の場合にのみウェブ参照を実行する:

- ユーザーが「最新情報もチェックして」「公式ドキュメントと照らし合わせて」等と明示的に指示した場合
- ユーザーが「スキルの仕様が変わったかもしれない」と懸念を示した場合

以下の場合はウェブ参照しない:

- 通常の構造チェック
- 新規スキル作成
- ローカルナレッジだけで回答できる質問

---

## 自己更新提案のフロー

ウェブ参照の結果、ローカルナレッジ（anthropic-guidelines.md や user-guidelines.md）との乖離を発見した場合:

1. 発見した乖離の内容を具体的に報告する
2. 「別セッションで skill-design-guide の更新を推奨します」と提案する
3. **現セッションでは skill-design-guide 自体の修正は行わない**

理由: ウェブ参照 + ナレッジ更新をチェック作業と同時に行うと、
コンテキストを圧迫し本来の作業品質が低下する。
更新は専用セッションで行うのが望ましい。

---

## 確認すべきポイントのガイダンス

ウェブ参照時に何を見るべきかのガイド:

### 公式ドキュメント確認時

- フロントマターのフィールド定義に変更がないか
- 新しい必須/推奨フィールドが追加されていないか
- Progressive Disclosure の推奨パターンに変更がないか
- 新しいディレクトリ構成パターンが推奨されていないか

### GitHub リポジトリ確認時

- 公式スキルの構造に新しいパターンがないか
- SKILL.md の書き方に新しいベストプラクティスがないか
- 廃止された機能や非推奨になった記法がないか
