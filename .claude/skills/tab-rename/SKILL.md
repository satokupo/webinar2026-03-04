# Tab Rename

タブ（tmux window）に、現在の作業内容を表す短い日本語タイトルを付けるスキル。tmux環境とVS Code統合ターミナルに対応。

## 前提条件

- tmux環境 または VS Code統合ターミナルでの使用を前提とする
- tmux: TMUX_PANEで特定されるウィンドウ（タブ）がリネーム対象（ウィンドウ切り替え後も安全）
- tmux: セッション名がデフォルト（数字連番）の場合、作業ディレクトリ名に自動変更する
- VS Code: 現在フォーカスのあるタブがリネーム対象
- VS Code内でtmuxを使っている場合はtmuxが優先され、tmuxのウィンドウ（タブ）名が変更される

## 処理フロー

### ステップ1: タイトル生成

会話コンテキストから、現在の作業内容を表す短い日本語タイトルを生成する。

**命名ガイドライン:**
- 日本語4〜10文字程度（最大12文字）、「何に対して何をしているか」が分かる省略形のタイトルにする
- 基本形は「対象＋動作」。助詞（の・を等）は省略推奨
- 対象には具体的な名前の省略形を使う（英語名は適切な日本語に変換）
- 対象が自明な場合は動作のみでも可
- 推奨例: 「リネームスキル改善」「認証API設計」「保存スキル修正」「命名ルール検討」「新スキル設計」
- 許容例: 「テスト実行」「リファクタ」「DB移行」（対象が自明）
- 悪い例: 「タブリネームスキルの命名ルール改善作業」（長すぎ）、「作業中」（対象も動作も不明）、「検討」（対象不明）、「スキル」（対象が曖昧）

### ステップ2: 環境判定とリネーム実行

スクリプトが環境を自動判定して適切な方法でリネームする。

#### tmux環境の場合（TMUX環境変数が存在）

警告・待機は不要。即座にスクリプトを実行する。

スクリプトはウィンドウ名変更に加え、tmux セッション名がデフォルト（0, 1, 2...の数字連番）の場合は
作業ディレクトリ名（例: dotfiles）に自動変更する。

```bash
python3 .claude/skills/tab-rename/scripts/rename_tab.py "<タイトル>"
```

#### VS Code環境の場合（TERM_PROGRAM=vscode）

⚠️ 警告を表示し、ユーザーの確認を得てからスクリプトを実行する:

```
⚠️🔒 タブ名を変更します。完了するまでタブを切り替えないでください！🔒⚠️
```

上記の警告を表示した後、`AskUserQuestion` ツールで確認を取る:
- question: 「タブ名を「<タイトル>」に変更します。対象のタブがアクティブですか？」
- header: 「タブ確認」
- 選択肢1: label「OK」/ description「現在のタブをリネームします」
- 選択肢2: label「キャンセル」/ description「リネームを中止します」
- multiSelect: false

**OK の場合** → 以下を実行する。
**キャンセルの場合** → リネームを実行せず、「リネームを中止しました」と報告して終了する。

セットアップチェック（初回のみ）:
```bash
python3 .claude/skills/tab-rename/scripts/setup_tab_rename.py
```

リネーム実行:
```bash
python3 .claude/skills/tab-rename/scripts/rename_tab.py "<タイトル>"
```

### ステップ3: 結果報告

スクリプトの出力をそのまま報告する。主な出力:
- タブ名変更: 「タブ名を「{title}」に変更しました」
- tmux セッション名変更（デフォルト名からの変更時のみ）: 「セッション名を「{dirname}」に変更しました」

## 重要な制約

- **軽量化**: タイトル生成にサブエージェント・ファイル読み込み・ログ解析等の重い処理を行わないこと。AIは既存の会話コンテキストから直接タイトルを判断する
- **追加の調査やファイル読み込みは一切行わない**
